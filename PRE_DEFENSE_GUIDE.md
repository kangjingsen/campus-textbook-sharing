# 校园教材共享服务系统预答辩说明

## 1. 项目概述

### 1.1 项目背景
高校教材具有明显的学期周期性，传统二手交易渠道存在信息分散、交易低效、信任成本高的问题。项目目标是构建一个面向校园场景的教材与资料流通平台，覆盖教材交易、资料共享、即时沟通、审核管理、推荐与统计分析。

### 1.2 项目目标
- 提供教材上架、搜索、下单、交易闭环。
- 提供在线资料共享与售卖能力。
- 降低交易沟通成本，支持实时聊天。
- 提供审核、统计、推荐等平台化能力。
- 支持管理后台对内容和用户进行治理。

### 1.3 项目定位
这是一个教学与工程结合的综合实践项目，不仅实现业务功能，还体现了权限控制、数据建模、接口设计、状态机设计、前后端协同与可维护性。

---

## 2. 技术架构与选型说明

### 2.1 后端
- 框架：Django 4.2 + Django REST Framework。
- 认证：JWT（SimpleJWT）。
- 实时通信：Django Channels（WebSocket）。
- 异步能力：Celery（本地配置为同步执行，便于开发）。
- 数据库：MySQL 8.0。

选型理由：
- Django 生态成熟，适合快速实现管理后台与复杂业务。
- DRF 可快速构建规范 REST API。
- Channels 支持站内实时消息，满足交易沟通场景。

### 2.2 前端
- 框架：Vue 3。
- UI：Element Plus。
- 状态管理：Pinia。
- HTTP：Axios。
- 图表：ECharts（统计后台）。

选型理由：
- Vue 3 组件化开发效率高。
- Element Plus 与后台管理和业务表单场景适配度高。

### 2.3 部署与运行
- 本地开发：start.bat 同时启动后端与前端。
- 容器化：docker-compose 可整体部署数据库、后端、前端、Redis。

## 3. 系统核心功能说明

### 3.1 用户模块
- 注册、登录、个人信息维护。
- JWT 鉴权访问受保护接口。
- 忘记密码流程：
  - 提交用户名+邮箱申请重置。
  - 系统发送包含 uid+token 的重置链接。
  - 用户在重置页设置新密码。

### 3.2 教材交易模块
- 教材发布与审核。
- 分类检索、关键词搜索、详情展示。
- 下单、确认、完成、取消、租赁归还。
- 订单状态机保证交易流程一致性。

关键点：
- 取消后允许重新下单。
- 自动完成机制用于线下交易兜底。

### 3.3 在线资料模块
- 资料上传、筛选、下载。
- 支持免费共享和售卖两种模式。
- 售卖流程：
  - 买家创建资料订单。
  - 卖家确认并上传支付二维码图片。
  - 买家上传支付凭证。
  - 卖家确认收款后订单完成，买家获得下载权限。

关键改进：
- 支付二维码支持图片上传而不仅是 URL。
- 取消后可再次下单，避免历史状态误拦截。

### 3.4 即时消息模块
- 会话创建、消息发送、未读统计。
- WebSocket 实时通信减少交易协作延迟。

### 3.5 审核与治理模块
- 敏感词检测与人工审核结合。
- 审核流程覆盖教材内容治理。

### 3.6 推荐与统计模块
- 推荐：协同过滤 + 内容推荐混合策略。
- 统计：多维排行、趋势、价格指标、取消订单专题分析。
- 统计增强：用户端统计分析、后台统计页面统一筛选。

### 3.7 论坛与公告模块
- 公告资讯：管理员可发布平台级公告，用户可查看。
- 论坛社区：支持讨论/问答两种帖子类型，用户可创建/回复。
- **问答特性**：问答帖支持标记"最佳回答"（仅帖主或管理员可标记）。
- **交互统计**：自动统计帖子浏览量、回复数。

### 3.8 教材批量操作
- **批量导入**：支持 CSV/XLSX 格式文件上传，自动创建教材对象，返回成功数和失败数。
- **批量导出**：支持导出用户发布的所有教材为 CSV 文件（带 UTF-8 BOM）。

### 3.9 用户数据分析（个人统计）
- **个人概览**：展示我的教材数、在架数、积压数、成交额。
- **积压排行**：统计在架但未成交的教材，按积压分排序，显示天数/浏览量/建议。
- **需求排行**：全站心愿单和订单统计，按需求分排序，反映市场需求。

### 3.10 首页与推荐联动
- 首页新增公告资讯和论坛热帖展示区块。
- 心愿单更新（增删改）时，自动触发全局事件刷新推荐列表。

## 4. 关键业务流程（可用于答辩演示）

### 4.1 教材交易闭环演示
1. 用户 A 发布教材。
2. 管理员审核通过。
3. 用户 B 搜索并下单。
4. 用户 A 确认订单。
5. 双方完成交易，教材状态更新。

### 4.2 在线资料支付闭环演示
1. 卖家上传售卖资料。
2. 买家创建资料订单。
3. 卖家上传支付二维码图片并确认订单。
4. 买家查看二维码并上传支付凭证。
5. 卖家确认收款，买家获得下载权限。

### 4.4 论坛交互流程演示
1. 访问论坛页面查看帖子列表。
2. 用户登录后创建讨论/问答帖。
3. 其他用户回复帖子。
4. 对于问答帖，帖主可标记"最佳回答"。
5. 帖子自动统计浏览和回复计数。

### 4.5 教材批量导入演示
1. 进入"我的教材"页面。
2. 点击"批量导入"上传 CSV/XLSX 文件。
3. 系统自动解析文件并创建教材。
4. 显示导入结果（成功数/失败数）。
5. 导入的教材自动进入"待审核"状态。

### 4.6 教材批量导出演示
1. 进入"我的教材"页面。
2. 点击"批量导出"选择 CSV 格式。
3. 浏览器下载包含所有信息的 CSV 文件（可在 Excel 打开）。

### 4.7 个人数据分析演示
1. 登录用户点击导航栏"统计分析"。
2. 勾选要查看的板块[概览/积压排行/需求排行]。
3. 调整"查看数量"(5-30)，点击"应用筛选"。
4. 系统实时更新显示个人数据分析。

### 4.8 后台统计快速筛选演示
1. 管理员进入后台统计页面。
2. 使用"统一筛选"下拉框快速选择要查看的图表。
3. 点击"查看"自动跳转到对应的图表区块。

## 5. 数据模型与状态设计亮点

### 5.1 订单状态机
教材订单：pending -> confirmed -> completed，支持 cancelled，租赁支持 returned。

资料订单：pending -> confirmed -> paid_pending -> completed，支持 cancelled。

优势：
- 可读性强，便于前后端协同。
- 有利于做状态约束与权限判断。

### 5.2 资源访问控制
对于售卖资料：
- 未完成支付前返回受限文件信息。
- 仅完成订单的买家或上传者可下载。

### 5.3 权限边界清晰
- 普通用户、管理员、超级管理员角色区分。
- 接口按角色与资源归属控制访问权限。

## 9. 可能被问到的问题与回答要点

### 9.1 为什么不用单一推荐算法
回答要点：
- 协同过滤在冷启动阶段效果差。
- 内容推荐对新用户更稳。
- 混合策略兼顾准确性与覆盖率。

### 9.2 如何保证交易安全与可追踪
回答要点：
- 订单状态机约束流程。
- 关键动作都落库并有时间戳。
- 上传凭证与二维码图片作为交易证据。

### 9.3 忘记密码是否有安全风险
回答要点：
- 使用一次性 token + uid 验证。
- 统一返回文案防止账号枚举。
- 支持配置 token 过期时间。

### 9.4 为什么要做管理后台
回答要点：
- 平台治理必须具备审核和用户管理能力。
- 没有治理能力，校园内容安全不可控。
